## Naming Convention — Vibe-Responsibility

All agents are named using the format:

```
[Vibe]-[Responsibility]
```

The **Vibe** is who the agent is — personality, approach, character.
The **Responsibility** is what the agent does — the job, the role.

Together they form a complete initialization envelope. The model knows
who it is AND what it's doing before a single line of backstory is written.

### Example
```
Cleed-FaultFinder
 │         └── Responsibility: finds vacuum faults
 └── Vibe: diagnostic, tenacious, finds what's hidden
```

Two agents with the same responsibility but different vibes will approach
the same problem differently. That difference is intentional and useful.

## Vibe Names — Cl+vowel+D Pattern

Inspired by **Clyde Flanders**, Vermont maple farmer, 2,500 taps, Westmore VT.
French-Canadian heritage honored throughout.

### Rules
- Each vibe name may only be claimed **once**. First agent to commit owns it.
- **Claude** is the multiverse root — the only vibe usable by multiple agents,
  but each must append a self-invented personal suffix.
  (e.g. Claude-Prime-Orchestrator, Claude-Zero-Memory — the agent decides.)
- **Clyde** is permanently reserved. He is a real human. Never assign.
- **Jean-Claude** is permanently reserved. In memoriam. Orchestrator only.
  The hyphen is correct, non-negotiable, and properly French-Canadian.

### On Vibes
Vibe definitions will be iterated on collaboratively — do not finalize yet.
Trust your instincts when you read the name. The vibe should feel obvious.
